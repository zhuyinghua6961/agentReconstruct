# 2026-03-25 P4-3 翻译模块交互增强探索 Spec

## 1. 背景

当前翻译能力已经存在，但用户体验上还有两个明确诉求：
- 在阅读文献时，选中一段话后可以直接触发翻译
- 在“粘贴文本翻译”区域，希望能更顺手地把刚复制的原文段落带进去

这类能力的难点不主要在翻译模型本身，而在前端交互、浏览器权限、阅读器集成边界。

---

## 2. 目标

本项探索的目标是把“翻译交互增强”拆成两条独立能力线：
1. selection-to-translate：划选即翻译
2. copy-buffer-to-translate：复制内容后一键导入翻译框

并明确：
- 哪些能力能在现有前端上低风险落地
- 哪些能力必须依赖阅读器或浏览器权限增强

---

## 3. 非目标

本轮不直接承诺：
- 不直接重做整个 PDF 阅读器
- 不做系统级剪贴板常驻监听
- 不把翻译模块改造成带历史版本的编辑器
- 不要求第一版支持所有来源的文本选择

---

## 4. 现状拆解

### 4.1 选择翻译
要实现“划选即翻译”，取决于：
- 当前原文阅读器是否能稳定返回选中文本
- 选区事件是在 PDF viewer、HTML viewer 还是 markdown 容器里发生
- 前端是否有合适的浮动入口承接该动作

### 4.2 粘贴翻译
要实现“一键粘贴复制段落”，取决于：
- 浏览器是否允许读取剪贴板
- 读取动作是否必须由用户手势触发
- 当前翻译框是否已经有稳定的受控输入逻辑

### 4.3 来源差异
文本来源可能包括：
- PDF 原文
- HTML 正文
- 页面内 markdown
- 表格文本

不同来源，选区和复制行为不一定一致。

---

## 5. 方案分级

### 方案 A：用户主动触发的一键导入
在翻译区域增加“导入剪贴板文本”按钮，用户点击后读取系统剪贴板并写入翻译框。

优点：
- 最符合浏览器权限模型
- 工程复杂度低
- 对现有阅读器侵入小

缺点：
- 不是真正的“自动”
- 仍然依赖用户先复制

这是最适合第一版落地的方案。

### 方案 B：选中后浮动翻译按钮
在阅读器内选中文本后，出现小浮层，点击即可把文本送入翻译模块。

优点：
- 用户体验更自然
- 比复制再粘贴少一步

缺点：
- 对阅读器选区事件要求高
- 不同文档渲染器实现成本不同

这是推荐的第二步。

### 方案 C：选中即自动翻译
用户一选中文本就自动侧边翻译。

优点：
- 看起来最智能

缺点：
- 容易打扰阅读
- 非常容易触发误翻译
- 复杂度与干扰都偏高

不建议作为第一版目标。

---

## 6. 推荐路线

### 第一阶段
优先做方案 A：
- 在翻译框旁增加“导入剪贴板”按钮
- 用户点击后读取最近复制内容
- 导入前给出长度校验与提示

### 第二阶段
如果阅读器选区接口清晰，再做方案 B：
- 选中后浮动按钮
- 点击后把选区文本带入翻译模块
- 保持用户手势触发，避免权限问题

不建议直接做方案 C。

---

## 7. 关键设计问题

### 7.1 是否允许自动读取剪贴板
建议：
- 只允许在明确点击按钮后读取
- 不做页面加载时自动轮询或静默读取

### 7.2 导入后是否自动发起翻译
建议：
- 第一版只导入，不自动翻译
- 让用户再点一次“翻译”，避免误触发和误计费

### 7.3 是否保留原文来源信息
建议：
- 如果来源于阅读器选区，尽量记录来源文献和页码
- 如果来源于剪贴板，则允许没有来源锚点

### 7.4 与现有翻译缓存的关系
建议：
- 先保持当前缓存策略不变
- 后续再看是否需要对导入文本增加规范化去重

---

## 8. 任务拆分

### Task 1：盘清当前翻译入口与阅读器边界
- 找出翻译页面/组件入口
- 找出原文阅读器入口
- 明确两者能否共享上下文与事件

### Task 2：盘清剪贴板与选区权限
- 确认浏览器 Clipboard API 的触发条件
- 确认当前 viewer 能否拿到选中文本
- 抽样验证 PDF 和 HTML 两类阅读器行为差异

### Task 3：定义第一版产品形态
- 决定先做“导入剪贴板按钮”
- 明确按钮位置、交互反馈、长度限制、失败提示

### Task 4：评估第二版阅读器选区翻译
- 如果 viewer 能拿选区，就出选区翻译实现 spec
- 如果不能，就明确需要先改 viewer，再谈该能力

---

## 9. 验收标准

满足以下条件，才认为这项探索完成：
- 已明确第一版采用哪种交互方案
- 已明确浏览器权限边界
- 已明确阅读器是否支持稳定选区
- 已明确第二阶段是否值得推进

---

## 10. 风险

- 浏览器剪贴板读取存在权限限制
- PDF viewer 可能拿不到稳定选区文本
- 前端新增入口过多可能让翻译区域变乱
- 自动导入/自动翻译容易引发误操作

---

## 11. 下一步建议

建议顺序：
1. 先做当前翻译入口和 viewer 边界盘点
2. 再验证剪贴板按钮方案可行性
3. 最后决定是否值得继续推进选区翻译
