# 主题体系：fastQA 向量库 vs patentQA 向量库（用于问题路由）

> **目的**：建立「工程/科研问题层级」的主题结构，支撑 **用户提问 → 选库 / 双库顺序** 的自动路由，**不做逐篇文献摘要**。  
> **依据**：仓库内对语料与检索形态的定义——fastQA 侧 `comparison_intent` 等对检索环境的描述（摘要/总结向量库 + MD/PDF 片段、LFP 及延伸主题）；patent 侧 `PatentCatalogRecord` / `PatentEvidence`（摘要、权利要求、说明书段落、表格补充、IPC/CPC 等）及双 Chroma 库（`patent_abstracts`、`patent_chunks`）的检索能力。

---

## 0. 两库在「可检索信息形态」上的根本分野（先读此表再读主题）

| 维度 | **fastQA** | **patentQA** |
|------|------------|--------------|
| 文档单元形态 | 论文 **DOI** 关联的摘要、章节切片、PDF/MD 片段；可叠加 **Graph KB** 行级事实 | **专利公开号** 锚定的摘要向量 + **说明书/权要邻近 chunk**；可带 **权利要求编号、段落号、表格结构化补充** |
| 叙述视角 | **学术共识、机理、实验趋势、模型解释** | **可实施技术方案、权利边界语言、实施例写法、分类号族内差异** |
| 定量信息 | 论文中的 **实验条件与结果量级**（与具体产线配方无绑定） | 实施例中常见的 **参数区间、配方列表、工艺步骤链**（与公开文本绑定） |
| 不可替代点 | **机理解释、材料学通识、跨论文综述型证据链** | **专利号级过滤、权要—实施例对齐、同族/分类号语境下的方案表述** |

以下主题粒度均控制在「可独立作为路由标签」的层级：不过宽（如不单列「电池」），不过细（不单列「某一次烧结曲线的一个温度点」）。

---

## 1. fastQA 向量库

### 【A】主题分类（结构化）

#### 主题 1：磷酸铁锂及相关正极材料的合成与粉体工程

| 字段 | 内容 |
|------|------|
| **主题名称** | 固相/液相/水热/溶胶凝胶等 **合成路线与粉体工程** |
| **主题说明** | 前驱体选择、锂源/铁源/磷源配比思路、一次颗粒形貌与二次颗粒团聚、干燥与煅烧对相纯度的影响等 **实验室—中试尺度** 问题。 |
| **常见子问题** | 「草酸亚铁路线和磷酸铁路线各有什么特点？」「碳包覆温度一般怎么选？」「一次颗粒纳米化对倍率有什么利弊？」 |

#### 主题 2：掺杂、包覆与表面改性

| 字段 | 内容 |
|------|------|
| **主题名称** | **掺杂、碳包覆、氧化物/聚合物包覆** 对电子/离子输运与稳定性的作用 |
| **主题说明** | 改性机理（导电网络、锂离子扩散路径、表面副反应抑制）与常见改性剂在 **文献实验语境** 下的对比。 |
| **常见子问题** | 「Mg 掺杂为什么能提高本征电子电导？」「软碳和硬碳包覆差异？」 |

#### 主题 3：电极浆料与涂布前段的流变与分散

| 字段 | 内容 |
|------|------|
| **主题名称** | **浆料流变行为**（粘度、触变性、分散性、固含量—粘度关系） |
| **主题说明** | 颗粒间相互作用、溶剂/粘结剂/导电剂对流变曲线的影响；偏 **机理与实验规律**，非产线 SOP。 |
| **常见子问题** | 「浆料中气泡对粘度的影响？」「固含量提高为什么粘度上升更快？」 |

#### 主题 4：电极结构与压实—孔隙—传输

| 字段 | 内容 |
|------|------|
| **主题名称** | **电极结构设计**（孔隙率、压实密度、曲折度与离子/电子传输） |
| **主题说明** | 极片微观结构与倍率、容量发挥、极化的关系；偏 **模型与实验关联**。 |
| **常见子问题** | 「压实密度和功率性能怎么权衡？」「孔隙率过低会怎样？」 |

#### 主题 5：锂离子扩散、极化与倍率/低温动力学

| 字段 | 内容 |
|------|------|
| **主题名称** | **离子扩散与电化学动力学**（扩散系数、交换电流、极化组成、倍率/低温限制因素） |
| **主题说明** | Olivine 结构一维扩散、颗粒尺寸效应、液相浓差极化等 **机理解释与文献数据范围**。 |
| **常见子问题** | 「LFP 倍率性能为什么差？」「纳米化主要缓解哪类极化？」 |

#### 主题 6：界面副反应与长期稳定性（含 SEI/CEI 语境）

| 字段 | 内容 |
|------|------|
| **主题名称** | **界面副反应、电解质相容性、产气与循环衰减机理** |
| **主题说明** | 在 **论文实验体系** 下讨论的界面演化、金属溶解、气体来源分析等。 |
| **常见子问题** | 「高温存储产气可能和什么反应有关？」「FEC 对 LFP 体系有什么用？」 |

#### 主题 7：全电池与系统集成语境下的 LFP 应用

| 字段 | 内容 |
|------|------|
| **主题名称** | **储能与动力电池应用** 中的 LFP 选型、与体系其它组件的匹配 |
| **主题说明** | 能量密度、成本、寿命、热管理在 **文献与综述** 中的论证方式。 |
| **常见子问题** | 「储能电站用 LFP 更关注哪些指标？」 |

#### 主题 8：回收、废水与资源化（条件敏感主题）

| 字段 | 内容 |
|------|------|
| **主题名称** | **回收再生、分离纯化、资源化** 与 LFP 产废链 |
| **主题说明** | 检索规划器明确：同一词在「制备」问法下可能是噪声，在「回收」问法下是 **正向主题**；路由时需 **意图词** 与制备类区分。 |
| **常见子问题** | 「退役磷酸铁锂粉怎么湿法回收锂？」 |

---

### 【B】fastQA 在各主题上的信息类型偏向

| 偏向维度 | 结论 |
|----------|------|
| **机理解释 vs 工程方案** | **整体偏机理解释与实验规律**；「怎么做产线」级 SOP **不是**该库的设计目标。 |
| **定量数据** | **包含**：论文中的条件—结果对应、典型量级范围；**不包含**：与特定工厂绑定的配方编号或内部工艺卡。 |
| **工艺 / 配方 / 参数** | **有**「实验室—论文表述」的工艺窗口与参数对比；**无**专利实施例那种 **权要语言 + 公开号绑定** 的写法。 |

## 2. patentQA 向量库

### 【A】主题分类（结构化）

#### 主题 P1：权利要求与保护范围解读（技术—法律交叉）

| 字段 | 内容 |
|------|------|
| **主题名称** | **权利要求技术特征分解与保护主题** |
| **主题说明** | 以 **权利要求书文本** 为单元的检索与对齐：哪些特征构成区别技术特征、从属权利要求的限定范围。 |
| **常见子问题** | 「权利要求 1 里限定的高温烧结时间范围意味着什么？」「从权 3 进一步限定了什么？」 |

#### 主题 P2：说明书实施例与参数化工艺链

| 字段 | 内容 |
|------|------|
| **主题名称** | **实施例：配方、工艺步骤、参数区间与效果表述** |
| **主题说明** | 说明书段落 + 表格补充中的 **多步工艺、组分列表、数值区间**；与 **专利公开号** 强绑定。 |
| **常见子问题** | 「有没有在 NMP 体系下高固含量仍低粘度的实施例？」「包覆两次 vs 一次的实施例对比？」 |

#### 主题 P3：与 LFP/锂电相关的材料制备与电极工艺（专利叙事）

| 字段 | 内容 |
|------|------|
| **主题名称** | **材料制备与电极制造** 在专利中的 **可实施披露** |
| **主题说明** | 球磨、喷雾干燥、包覆、涂布、辊压等在 **实施例与权要** 中的组合写法；强调 **可重复实施** 而非学术模型。 |
| **常见子问题** | 「提高压实密度的专利一般从哪些结构特征入手？」 |

#### 主题 P4：摘要与标题层面的技术路线检索

| 字段 | 内容 |
|------|------|
| **主题名称** | **摘要向量检索 — 技术路线快速定位** |
| **主题说明** | `patent_abstracts` 用于 **粗定位** 相关专利族与技术方向，再下钻 chunk。 |
| **常见子问题** | 「有哪些专利涉及硅碳复合包覆 LFP？」（偏检索入口） |

#### 主题 P5：分类号（IPC/CPC）与申请人语境下的方案族

| 字段 | 内容 |
|------|------|
| **主题名称** | **分类号与申请人维度的方案聚类** |
| **主题说明** | 元数据中带 **IPC/CPC、申请人** 时的族内对比、技术路线跟随。 |
| **常见子问题** | 「H01M 10/0525 下常见的正极浆料分散方案有哪些？」 |

#### 主题 P6：表格与结构化实施数据

| 字段 | 内容 |
|------|------|
| **主题名称** | **表格化实施例数据**（组分、工艺参数、性能对照） |
| **主题说明** | `PatentTableSupplement` 类路径：**列名—行字典** 形态，适合「多实施例数值对比」类问题。 |
| **常见子问题** | 「实施例表里哪几组固含量—粘度最优？」 |

---

### 【B】patentQA 在各主题上的信息类型偏向

| 偏向维度 | 结论 |
|----------|------|
| **机理解释 vs 工程方案** | **整体偏工程方案与可实施披露**；机理解释常服务于 **支持技术效果** 的说明书表述，而非独立学科模型。 |
| **定量数据** | **强包含**：实施例与表格中的 **区间、列表、对照组**；与 **公开文本** 绑定，便于溯源到专利号。 |
| **工艺 / 配方 / 参数** | **核心覆盖区**：步骤链、配方、设备/条件用语与 **权利要求语言** 一致化程度高。 |

## 3. 【C】主题覆盖差异（fastQA vs patentQA）

### 3.1 两者都有，但侧重点不同（必须看差异）

| 联合主题 | **fastQA 侧重** | **patentQA 侧重** |
|----------|-----------------|-------------------|
| **浆料流变与分散** | 机理、分散模型、文献中的粘度—固含量规律 | 实施例中的 **分散剂体系、搅拌/涂布条件、配方** |
| **压实密度 / 孔隙 / 电极结构** | 结构—传输—性能关系的 **机理解释与论文数据** | **权要 + 实施例** 中的结构特征组合、辊压/级配写法 |
| **包覆 / 掺杂改性** | 改性机理、文献中的改性剂对比实验 | **可实施配方与工艺窗口**、与现有技术的区别表述 |
| **倍率 / 循环 / 低温** | 动力学限制因素、颗粒尺寸与电解液匹配的 **学术分析** | 实施例中的 **全电池或半电池条件、对比表** |
| **合成路线（固相/水热等）** | 反应路径、相形成、杂质控制 **机理** | **工业可操作步骤**、设备与参数链、权利要求布局 |

### 3.2 主要集中在 fastQA

- **离子扩散与极化机理的学科化解释**（一维扩散、交换电流、浓差极化分解）。  
- **界面副反应的一般电化学与谱学论证语境**（以论文体系为主）。  
- **跨多篇论文的综述型对比**（非单族专利对比）。  
- **回收/资源化链** 在「用户明确问回收」时的 **学术与工艺综述**（与制备问法需意图区分）。

### 3.3 主要集中在 patentQA

- **权利要求解读、保护范围、从属限定**。  
- **专利号级过滤与权要—chunk 对齐检索**。  
- **实施例表格的多组参数对照**。  
- **IPC/CPC/申请人族内的方案检索与规避设计语境**（若业务启用对应元数据）。

## 4. 路由建议（按主题 → 库选择 / 是否双库）

规则写法：**单箭头** = 主检索库；**+** = 固定顺序双库（前者先建立认知或证据，后者补工程/专利披露）。

| 主题（工程/科研层级） | 更推荐 | 是否双库联合 |
|------------------------|--------|----------------|
| 浆料流变行为（粘度、触变性、分散性） | fastQA | **fastQA → patentQA**（机理 + 调浆/分散实施例） |
| 电极结构设计（孔隙率、压实密度） | fastQA 起笔 | **fastQA → patentQA**（结构—性能机理 + 辊压/级配专利披露） |
| 离子扩散与动力学 / 倍率限制 | fastQA | 单库即可；若问句明确要求「工业上怎么改颗粒级配」→ **+ patentQA** |
| 界面副反应（SEI/CEI）机理解释 | fastQA | 单库；若涉及「产线化成产气专利对比」→ **+ patentQA** |
| 材料制备工艺（球磨、喷雾、包覆）— **机理与路线对比** | fastQA | 若含 **配方/参数窗口/如何工业化** → **patentQA 优先**，可前接 fastQA 机理 |
| 材料制备— **可实施配方与参数** | **patentQA 优先** | 可选 **fastQA → patentQA**（先学术共识再专利实施例） |
| 掺杂/碳包覆改性机理 | fastQA | 若问「哪些包覆剂组合在专利中常见」→ **patentQA 优先** |
| 权利要求 / 无效 / 规避 / 同族对比 | **仅 patentQA** | 否 |
| 回收/资源化（制备语境易混） | fastQA（意图=回收时） | 若需 **实施装置/工艺专利** → **+ patentQA** |
| 储能/动力应用场景选型 | fastQA | 若问 **某公司产品专利布局** → **patentQA** |

**编码落地提示**：「双库联合」在编排层应输出 **有序 steps**，与 `docs/auto-retrieval-routing-fastqa-vs-patent.md` 中的路由枚举对齐；主题层标签可作为 **先验权重** 叠加在规则引擎之上（例如对「浆料流变」默认附加 patent 第二步）。

## 5. 维护与校准

- **语料扩展**（新增 IPC 段、新专利族、fastQA 新集合）时：只增删 **【A】主题块** 与 **【C】差异表** 的行，不改变 **0 节分野表** 的逻辑结构。  
- **禁止**用本文件替代法律意见：涉及 FTO、侵权结论的 **最终决策权** 不在向量检索层。
